# Fluxo de Entrega

## Sequencia oficial

`spec -> plan -> tasks -> implementacao -> validacao -> documentacao -> versionamento`

## Regras

1. Toda feature nova ou mudanca estrutural deve nascer em `specs/`.
2. `spec.md` define o que sera entregue.
3. `plan.md` define abordagem tecnica, restricoes e impactos.
4. `tasks.md` organiza a execucao em incrementos verificaveis.
5. A entrega nao termina sem documentacao sincronizada e contexto vivo atualizado.
